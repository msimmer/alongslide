Alongslide is a responsive, parallax-inspired presentation framework for long-form reading on the web, employing a lightweight templating syntax for Markdown. »

The various components of Alongslide will be open-sourced by early 2014.

Technology layers

  1. A fine-tuned parallax-like scrolling framework built on a fork of Skrollr
  2. Flowing text columns using a custom implementation of the emerging CSS Regions spec
  3. A responsive-and-beyond layout logic in CSS+JS.
  4. A lightweight templating syntax for Markdown implemented in Ruby with Redcarpet and Treetop

Alongslide puts layout first, respecting the aspect ratio of the content area at all window sizes and resolutions.

On desktop, hit opt-g to see the grid(s)—and resize the window to any size or shape.

Try it on mobile, too.

In addition to normal flowing text, Alongslide offers two main structural elements:

  • Sections break up flowing text sequentially
  • Panels function like sidebars which can interrupt or supplement the flowing text.

In Alongslide Markdown, these structural elements are specified by indented blocks of text (traditionally used for <code>) to show visually that they sit outside the primary flow.

Both sections and panels may specify transitions—either fading or sliding in.

Panels may occupy the full content area, or be pinned to one edge—being unpinned at a later point in the flowing text. Their size may also be configured as a simple one-word parameter.

Panels may contain user-specified templates, which may be nested according to their defined “greediness”. Templates defined as “greedy” will effectively consume any subsequent Markdown, treating it as their contents. “Non-greedy” templates will treat subsequent Markdown as a sibling.

As with Markdown, a philosophical principle of Alongslide is that WYSIWIG authoring environments are not as powerful as a simple enough syntax enabling (and encouraging) writers and editors to write solid, structural markup.
